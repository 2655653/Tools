## 一、大麦抢票脚本的技术原理

### 核心架构

典型的大麦抢票脚本基于 Python 开发，主要模块包括：

| 模块 | 作用 |
|------|------|
| **HTTP 请求层** | 用 `requests` 库发送请求，模拟选票、加购物车、提交订单 |
| **浏览器自动化层** | 用 `Selenium` / `Playwright` 控制真实浏览器，模拟点击操作 |
| **网页解析层** | 用 `BeautifulSoup` / `lxml` 解析演出页面元素 |
| **验证码识别层** | 对接第三方打码平台 API（云打码、打码兔等）识别图形验证码 |
| **登录持久化层** | 用 `pickle` 保存 Cookie，实现免重复登录 |

核心流程为：

```python
# 伪代码示例（参考公开教程思路）
1. 读取已保存的 Cookie → 保持登录状态
2. 访问目标演出页面 → 获取演出ID、场次ID、票价ID
3. 循环尝试购票:
   a. 发送选票请求 (select_ticket)
   b. 识别验证码 (若出现)
   c. 加入购物车 (add_to_cart)
   d. 提交订单 (submit_order)
   e. 确认购票人信息
4. 等待支付 → 用户手动完成支付（大多数脚本不支持自动支付）
```

## 二、大麦网的反爬虫机制（这是最大的难点）

根据最新资料，大麦网在 **2025年已经部署了极强的防护体系**：

### 第一层：动态签名加密

- 采用 **HMAC-SHA256 多级签名**，每次请求都需要实时生成签名
- 签名密钥 **每小时变化**，必须实时同步获取
- 关键请求头包括：
  - `X-Sign` — 动态签名
  - `X-T` — 毫秒级时间戳
  - `X-Device-ID` / `X-UMID` — 设备指纹

### 第二层：Native 层防护

- 核心加密逻辑在原生 SO 库中（`libmtguard.so` / `libsgmain.so` / `libvmp.so`）
- 需要逆向分析这些库才能获取签名算法，**普通 Python 脚本已无法直接绕过**

### 第三层：设备指纹 + 行为检测

- 收集设备信息、浏览器指纹、操作行为模式
- 能识别出虚拟机、模拟器或异常环境

### 第四层：下单/支付环节风控

- 即使请求成功到达服务器，下单时还会进行多重风控校验

## 三、可行方案分析

### 方案 A：完全自研脚本（⚠️ 不推荐）

自己写 Python 脚本直接调用接口。

- **优点**：灵活、可定制
- **致命缺点**：大麦现在的签名机制极其复杂，需要逆向 Native 库，普通人根本无法实现。公开的"简易教程"绝大多数在 2025 年后已经 **完全失效**。

### 方案 B：浏览器自动化（Selenium / Playwright）⭐ 推荐

模拟真实用户操作，用浏览器自动点击。

```python
# 思路示例
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch(headless=False)  # 建议非无头模式
    page = browser.new_page(
        viewport={"width": 1280, "height": 720},
        user_agent="Mozilla/5.0 (Windows NT 10.0; Win64; x64) ..."
    )
    # 1. 登录
    page.goto("https://www.damai.cn/")
    # ... 执行登录操作 ...
    # 2. 访问演出页面
    page.goto("https://detail.damai.cn/item.htm?id=XXX")
    # 3. 循环点击购票按钮
    while True:
        try:
            page.click(".buybtn")
            break
        except:
            page.wait(0.1)
```

- **优点**：模拟真实用户，绕过接口签名问题；`Playwright` 比 `Selenium` 更快更稳定
- **缺点**：速度比直接发请求慢；仍需应对滑动验证码

### 方案 C：借助现成开源项目 ⭐⭐ 最推荐

目前 GitHub 上有一些维护中的抢票项目（如 `Automatic_ticket_purchase` 等），它们会随着大麦的更新而更新。使用前需：

1. 先看 **最近更新时间**，选择活跃维护的项目
2. 按照 README 配置 Cookie、演出 ID、购票人信息
3. 准备好 **打码平台 API**（用于识别验证码）

> **但请注意**：这些项目也可能随时因大麦升级而失效。

### 方案 D：加速浏览器 + 辅助插件

不需要写脚本，用浏览器插件解决部分问题：

- **Auto Refresh** 插件 — 自动刷新演出页面
- **Tampermonkey** + 用户脚本 — 自动点击购票按钮
- **秒杀助手**类浏览器插件 — 自动填写信息

## 四、实战攻略清单

### 准备阶段

| 步骤 | 操作 | 重要性 |
|------|------|--------|
| 1 | 提前在大麦 App/网页登录好账号 | ⭐⭐⭐ 必须 |
| 2 | 提前添加好观演人（最多选 2-4 人） | ⭐⭐⭐ 必须 |
| 3 | 提前设置好默认收货地址 | ⭐⭐⭐ 必须 |
| 4 | 提前在购物车清空其他演出 | ⭐⭐ 建议 |
| 5 | 准备好支付方式（余额宝/银行卡需提前充值） | ⭐⭐⭐ 必须 |
| 6 | 注册打码平台账号（如需） | ⭐ 可选 |
| 7 | 找一个 **网速快、网络稳定** 的环境 | ⭐⭐⭐ **最关键** |

### 抢票当天的操作要点

1. **提前 5-10 分钟**进入演出页面，反复刷新直到购票按钮可点击
2. 脚本/插件提前设好目标场次和票价
3. 开票后 **不要犹豫**，立即点击
4. 验证码出现时，打码平台要秒级响应
5. 选座模式下，尽量选 **靠过道/靠前排** 的座位
6. 加入购物车后 **立即提交**，不要在购物车停留太久

## 五、关键提醒与风险提示

> **⚠️ 法律与账号风险**
> - 大麦明确禁止使用自动化工具抢购，违规可能导致**账号被封禁**、**购票资格被取消**
> - 使用第三方脚本存在**账号泄露风险**（特别是需要提供 Cookie 或账密的），请务必甄别代码来源
> - 抢票脚本本质上是"有条件地绕过平台规则"，请自行承担风险

> **⚠️ 性价比提醒**
> - 如果你有一定编程基础，方案 B（Playwright 自动化）或方案 C（开源项目 + 打码平台）是最可行的路径
> - **脚本不能保证 100% 抢到**，再好的脚本也受网络延迟、打码速度、目标票数的影响
> - 有时**找代抢（人工帮忙抢）**反而比你花大量时间写脚本更实际

## 七、实际生成的代码包

**项目路径**：`c:\Users\tang'wen'jing\Desktop\这次一定有票\damaihelper\`

基于 GitHub 最活跃项目 [damaihelper](https://github.com/Guyungy/damaihelper) v2.5.0（2026.3.30）改编，包含以下文件：

### 7.1 核心脚本 `ticket_script.py`

完整的大麦抢票自动化脚本，核心类 `Concert` 包含：

| 方法 | 作用 |
|------|------|
| `get_cookie()` | 手动扫码登录，保存 Cookie |
| `set_cookie()` | 加载已保存的 Cookie |
| `login()` | 进入演出页面并注入 Cookie |
| `enter_concert()` | 初始化 Chrome 浏览器（反检测 + 手机端模拟） |
| `choose_ticket()` | 核心抢票逻辑：选日期 → 选场次 → 选票价 → 加购 |
| `check_order()` | 选择观演人 → 提交订单 → 跳转支付页 |

浏览器启动参数说明：
- `--disable-blink-features=AutomationControlled` — 隐藏 webdriver 痕迹
- `mobileEmulation: Nexus 6` — 模拟手机端视图，更稳定
- `pageLoadStrategy: eager` — 不等待页面完全加载，提升速度
- 禁止图片/CSS 加载 — 进一步加速

### 7.2 配置文件 `config.json`

```json
{
  "nick_name": "你的大麦昵称",
```

### 7.3 一键启动脚本

- **`环境安装.bat`** — 安装 Python 依赖 + 检测 Chrome 版本
- **`一键启动抢票.bat`** — 启动抢票脚本

详细使用说明见：`damaihelper/README使用指南.md`

### 7.4 使用流程

```
1. 下载 chromedriver.exe（版本需与 Chrome 匹配）→ 放入 damaihelper 目录
2. 填写 config.json（演出 URL、观演人序号等）
3. 双击"环境安装.bat" → 安装依赖
4. 首次运行 python ticket_script.py → 弹出浏览器 → 扫码登录 → Ctrl+C 停止
5. 双击"一键启动抢票.bat" → 等待开票 → 自动抢票 → 手动支付
```
