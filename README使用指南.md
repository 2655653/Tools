## 三、配置 config.json（⚠️ 最重要步骤）

用记事本或 VS Code 打开 `config.json`，按实际情况填写：

```json
{
  "nick_name": "你的大麦昵称",
  "date": [1, 2],
  "sess": [1, 2],
  "price": [1, 2],
  "real_name": 1,
  "ticket_num": 2,
  "viewer_person": [1, 2],
  "damai_url": "https://www.damai.cn/",
  "target_url": "https://detail.damai.cn/item.htm?id=你的演出ID",
  "driver_path": "chromedriver.exe"
}
```

### 字段说明

| 字段 | 说明 | 示例 |
|------|------|------|
| `nick_name` | 你的大麦账号昵称 | `"麦子"` |
| `date` | 日期优先级（从1开始，支持多选备选） | `[1, 2]` |
| `sess` | 场次优先级（如第1场、第2场） | `[1]` |
| `price` | 票价档位优先级（数字越小优先级越高） | `[1]` |
| `real_name` | 实名者序号 | `1` |
| `ticket_num` | 购买几张票 | `2` |
| `viewer_person` | 观演人序号（提前在大麦添加好） | `[1, 2]` |
| `target_url` | **目标演出页面网址** | 见下方教程 |
| `driver_path` | chromedriver 路径 | `"chromedriver.exe"` |

### 如何获取 `target_url`（演出ID）

1. 打开大麦网：https://www.damai.cn
2. 找到你要抢的演出，点击进入
3. 复制浏览器地址栏的完整 URL，类似：
   ```
   https://detail.damai.cn/item.htm?id=812345678
   ```
4. 把这段 URL 填入 `target_url`

## 六、脚本状态码说明

| status | 含义 |
|--------|------|
| 0 | 初始状态 |
| 2 | 检测到"即将开抢"，等待中 |
| 3 | 选座购买模式（需手动选座） |
| 4 | 提交订单成功，等待确认 |
| 5 | 选座购买，等待用户选座 |
| 6 | 订单已提交，等待支付 |

## 七、异常处理机制

脚本内置多层异常保护：

1. **浏览器被关闭** → 暂停打印提示 → 等待用户按回车 → 自动重启浏览器继续抢票
2. **页面元素找不到** → 自动返回详情页 → 重新尝试
3. **加号按钮找不到** → 判断按钮类型（选座购买/提交缺货登记）→ 相应处理
4. **超时不跳转支付页** → 询问用户选择（确认成功 / 重新抢票）
5. **任何异常** → 打印错误 → 尝试返回详情页 → 继续下一轮

按 `Ctrl+C` 可随时安全中断脚本。

## 八、调试与常见问题

### Q1：提示 chromedriver 相关错误

脚本使用本地缓存路径：
```
C:\Users\tang'wen'jing\.wdm\drivers\chromedriver\win64\146.0.7680.165\chromedriver.exe
```
如有问题，查看 Chrome 版本（`chrome://settings/help`），从镜像站下载替换。

### Q2：脚本运行了但一直"刷新等待"

- 可能还没到开票时间，脚本在等待按钮可点击
- 检查 `config.json` 中的 `target_url` 是否正确
- 检查 `date`、`sess`、`price` 序号是否超范围

### Q3：提示 "cookie 加载失败"

删除 `cookies.pkl`，重新运行脚本并扫码登录。

### Q4：页面元素找不到（DOM 结构变化）

大麦网可能随时改版，XPath 可能失效。
解决方案：按 F12 打开开发者工具，重新定位按钮元素并修改脚本中的 XPath。

### Q5：速度不够快，抢不到票

- 脚本的速度受限于 Selenium 的响应时间
- 建议同时开启"手动刷新 + 脚本辅助"双保险
- 选择网络延迟最低的上网环境（公司网络 / 机房专线更快）

## 九、进阶优化方向

如果基础脚本效果不够好，可以考虑：

1. **升级到 Playwright**（比 Selenium 更快、更稳定）
2. **对接打码平台 API**（自动识别验证码）
3. **多线程并发**（同时开多个浏览器窗口抢不同座位）
4. **Android 真机 + Appium**（最稳定，但需要真机）
